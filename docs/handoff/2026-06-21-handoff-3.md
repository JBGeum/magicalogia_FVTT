## 4. 함정 / 참고

- **커밋 메시지**: 영어 한 줄 conventional, **co-author 없음**. lint-staged(prettier/eslint)가 커밋 시 md·scss·hbs·mjs 자동 정렬(정상).
- **이번 세션 결정**:
  - `buildChargeCard`의 `spent` 필드는 **미사용이지만 시안 헬퍼 파리티로 유지**(사용자 결정 — 결함 아님).
  - max는 모델 `charge`의 `max:6` 고정 = `CHARGE_SLOTS`(actor-sheet). **호출부 전달**(신규 공유 상수 도입 안 함).
  - 루비(`nameRt`)는 spell 모델에 필드 없어 **제거**(이름만). 필요 시 모델 필드 추가는 별도.
  - `delta=0`("▲ +0")은 `#onSetCharge`의 `after===before` 가드로 카드까지 **도달 불가**.
  - `charge-card.hbs`의 `.lbl`은 비-BEM이지만 시안 CSS `.mg-cc__charge .lbl` 패턴 그대로(Task3 SCSS가 `.mg-cc__charge` 아래 중첩) — 고아 아님.
- **시안 토큰 이식 규칙**: 시안 CSS의 `--mg-*`/`.theme-*`는 이식 금지(우리 `scss/theme/_tokens.scss` 재사용). 컴포넌트 규칙만 `.magicalogia { }` 중첩 SCSS로. 이번 신규 색 토큰 **0**(`--mg-good*`/`--mg-bad*` 등 기존 재사용).
- **데이터 모델 변경 없음**(`charge` 필드 기존). 채팅 카드는 **라이트 고정**.
- **신규 standalone 채팅 카드**는 `templates/chat/`에 두고 `helpers/templates.mjs` `loadTemplates`에 등록. 발행은 `spell-cast.mjs`의 `castSpell`/`spell-charge.mjs`의 `postChargeCard`처럼 `renderTemplate`+`ChatMessage.create`.
- **순수 로직은 Foundry 비의존 모듈로 분리해 단위 테스트**(`buildChargeCard`/`resolveSpecialtyTn`/`formatCost`/`specialtyGrid`/`classifyRoll`/`renderDie`). Foundry 의존(`postChargeCard`/`castSpell`/시트/앱)은 F5 육안.
- **서버 재시작 vs F5**: 모델/`static TABS`/액션 등록/settings 변경은 서버 재시작 필요할 수 있음. 템플릿/SCSS만이면 F5.
- **SDD 진행 로그(복구용)**: `.superpowers/sdd/progress.md` — 이번 기능 task별 커밋·리뷰·Minor 전부 기록. (gitignore 스크래치, `git clean -fdx` 시 소멸 — 그 경우 `git log`로 복구.)
- **Write로 md 생성 시 파일 끝 도구 태그 새어듦** 주의 — `tail`로 확인.
